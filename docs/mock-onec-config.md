# mock-onec-config.json

`mock-onec-config.json` содержит полностью вымышленные данные.

Эти данные нельзя использовать как пример данных 1С. Файл нужен только для автономного smoke-теста Go-бота без опубликованной 1С.

- `mock-onec-config.json` не является источником данных. Все значения внутри него нужны только для автономного smoke-теста Go-бота без 1С.
- Реальный `account_id` должен возвращаться 1С из базы при `/max/v1/account-link/confirm`.
- Реальный `meter_id` должен возвращаться 1С из базы при `/max/v1/accounts/{account_id}/meters`.
- Реальные начисления, услуги, тарифы, приборы, квитанции, обращения и записи должны браться из базы 1С.
- Значения из mock-файла нельзя переносить в production-настройки, README для реальной 1С или пользовательские инструкции.
